**3. Discovery Value**

- Created detailed debt stories

### What We Learned

**1. Item Type System**
- PRODUCT: Standard inventory tracking
- SERVICE: No inventory, revenue only
- INGREDIENT: Raw materials for recipes
- RECIPE: BOM composition (schema only, no implementation)

**2. Pricing Complexity**
- Fallback logic: outlet → company → null
- Virtual scope_key for uniqueness
- Bulk pricing updates
- Price history not tracked (gap identified)

**3. Technical Debt Patterns**
- Schema complete but features missing
- Recipe composition not implemented
- COGS posting not integrated
- Cost tracking settings exist but no implementation

### Challenges

**1. Recipe/BOM Gap**
- Item type exists
- No recipe_ingredients table
- No cost calculation
- No UI for recipe management

**2. Accounting Integration**
- Items don't link to COGS accounts
- No automatic COGS posting on sale
- Cost tracking incomplete

**3. Advanced Features Missing**
- No item variants (size, color)
- No barcode support
- No product images

## 📝 LESSONS LEARNED

### For Technical Architecture
1. **Pricing Models:** Company defaults + outlet overrides is powerful
2. **Item Types:** Schema should include feature flags (track_stock, has_recipe)
3. **Accounting Link:** Items should reference COGS accounts from start

### For Discovery Process
1. **Debt Identification:** Audits should flag incomplete features
2. **Spec Creation:** Write specs for debt immediately while context fresh
3. **Priority Setting:** Distinguish core from enhancements

### For Project Planning
1. **Epic Scope:** 85% complete is "done" for core functionality
2. **Debt Tracking:** Create separate stories for technical debt
3. **Pilot Readiness:** Core complete = pilot ready, debt = polish
