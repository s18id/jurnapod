### What We Learned

**1. Cross-Cutting Concerns Require Explicit Attention**
- Every story needed code review fixes (4/4 = 100%)
- Issues found: missing telemetry, wrong storage mechanism, missing test files
- **Lesson:** Add cross-cutting concerns checklist to story template

**2. Epic 8 Retro Commitments Were Not Tracked**
- PageHeader component: Not done (carryover to Epic 10)
- ActionDropdown extraction: Not done
- **Lesson:** Convert retro commitments to sprint tasks immediately

**3. Brownfield Complexity Underestimated**
- users-page.tsx was 1,300+ lines
- Required significant reading/understanding before changes
- **Lesson:** Budget time for architecture comprehension in estimates

**4. Non-Functional Requirements Not Explicit in ACs**
- Telemetry mentioned but not as first-class concern
- URL state persistence required code review to catch
- **Lesson:** Make NFRs explicit in acceptance criteria

### Challenges

**1. Code Review Fixes Required for Every Story**
- 8 total issues found (2 Critical, 4 High, 2 Medium)
- All resolved, but pattern indicates process gap
- **Action needed:** Improve pre-review quality

**2. Telemetry Implementation Incomplete**
- Used console.log for development
- Not production-ready observability
- **Action needed:** Define telemetry API contract

**3. Documentation Gaps**
- No architecture decision log
- Design decisions not documented
- **Action needed:** Create ADR for backoffice patterns
